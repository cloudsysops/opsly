# OpenClaw — Configuration for Opsly

## Overview

**CLI `openclaw` (global):** requiere **Node.js ≥ 22.12**. En la raíz del repo, `nvm install && nvm use` respeta `.nvmrc` (Node 22). El monorepo y CI siguen siendo compatibles con Node 20+ para `npm run type-check`, etc.

OpenClaw es el **framework de orquestación multi-agente** de Opsly. Utiliza:
- **MCP** como punto de entrada de herramientas
- **BullMQ** para colas de jobs con prioridad por plan
- **LLM Gateway** para routing y caché de LLMs
- **Context Builder** para mantener contexto entre sesiones

## Stack

| Componente | Path | Puerto | Rol |
|------------|------|--------|-----|
| MCP Server | `apps/mcp` | 3003 | Herramientas para agentes externos |
| Orchestrator | `apps/orchestrator` | 3011 | Cola BullMQ + workers |
| LLM Gateway | `apps/llm-gateway` | 3010 | Cache, routing, costos |
| Context Builder | `apps/context-builder` | 3012 | Sesión + resúmenes |

## Workflow

```
Agente → MCP Tools → Orchestrator (encola) → Workers (ejecutan) → LLM Gateway (opcional)
```
